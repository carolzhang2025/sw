Automatic Universal Gateway

Empowering IT/OT Synergy with Standardized Data Connectivity and Web APIs.

What is AUG ?

AUG (Automatic Universal Gateway) is an industrial communication integration platform that can integrate various Industrial communication protocols. It connects PLCs, dashboards, and SCADA systems to enable real-time data exchange and seamless database integration.

AUG supports many popular database servers such as Modbus, OPC DA/UA, DDE Server, PI System, SQL databases, Web APIs, and widely-used protocols like Siemens, Emerson, Omron, Allen-Bradley,Mitsubishi, Yaskawa, Keyence and more, simplifying data flow between devices and enterprise systems while reducing driver management overhead. In addition to protocol conversion, AUG provides Web API interfaces for IT developers to build custom applications, laying a solid foundation for smart factory communication and data collection.

How AUG works

AUG collects data from field devices, control systems, and various industrial communication protocols, then converts it into standardized and structured data formats, allowing upper-level systems such as PI system, Web API, and OPC to easily read and analyze the data.

You can also leverage the database as a simulator data source and use AUG as a simulator to streamline system testing.

01

Comprehensive Field Connectivity

Seamlessly capturing raw data from PLCs, instrumentation, and diverse industrial assets to build a robust production baseline.

02

Industrial Protocol Transformation

Utilizing AUG - I/O Adapters to convert diverse industrial protocols into standardized and structured data formats for universal interoperability.

03

Centralized Hub & Simulation

Managing core data storage through AUG - Data Server, while leveraging the database as a simulator source to streamline system testing and development.

04

Seamless System Integration

Transforming field data to enable upper-level systems (such as PI System, Web API, and OPC) to easily read and analyze information.

Customer story

From Scattered to Centralized : Achieving Plant-Wide HVAC Energy Monitoring with AUG

Manufacturing in Taiwan faces rising pressure to decarbonize. With HVAC systems accounting for 30-40% of total energy usage, they are a primary target for savings. However, fragmented systems and data opacity often hinder effective energy management.

A leading Taiwan-listed manufacturer with a 70-year legacy, our client operates across Asia, specializing in semiconductor lead frames and metal stationery. To achieve its sustainability goals, the company has prioritized its energy-intensive HVAC systems for the first phase of digital upgrades.

Technical barriers stalled the company’s push for energy efficiency :

  • Data Acquisition Obstacles : Dispersed Systems

    The HVAC systems are scattered across the facility, making manual data gathering highly inefficient and hindering comprehensive monitoring.

  • Integration Bottlenecks : Connectivity Gaps

    The facility lacked the necessary tools to bridge the gap between field-level PLCs and the SCADA system, preventing real-time data visibility.

The lack of data transparency created a blind spot in energy management, stalling the transition to optimized efficiency.

To overcome these integration hurdles, the company deployed AUG as the central intelligence hub, bridging the gap between field hardware and management software :

  • Step 1 : Unifying Distributed Systems

    By using AUG as the central gateway, all distributed PLCs were connected to a single platform, consolidating every data point into a unified communication stream.

  • Step 2 : Seamless Protocol Conversion

    With AUG’s built-in modules, data collected from PLCs is instantly converted into SCADA-ready formats (e.g., Modbus/MELSEC to OPC DA)—no programming skills required.

By deploying AUG as the central intelligence hub, the company successfully bridged the hardware integration gap :

  • Centralized Management :

    Seamlessly converted protocols (Modbus/MELSEC to OPC DA) for SCADA compatibility without requiring any programming skills.

  • Seamless Protocol Conversion :

    With AUG’s built-in modules, data collected from PLCs is instantly converted into SCADA-ready formats (e.g., Modbus/MELSEC to OPC DA)—no programming skills required.

The implementation of AUG not only resolved data collection challenges but also established a rock-solid digital foundation, driving the enterprise's transition from traditional management to data-driven, intelligent energy-saving strategies.
